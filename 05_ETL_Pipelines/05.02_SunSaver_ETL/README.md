## 01 В· DescripciГіn del proyecto

SunSaver es un pipeline ETL de producciГіn que ingiere datos meteorolГіgicos en tiempo real (OpenWeatherMap), precios horarios del mercado elГ©ctrico espaГұol (REE/PVPC) y parГЎmetros de instalaciГіn por cliente, para generar **predicciones de generaciГіn fotovoltaica y consumo industrial** en un horizonte de 5 dГӯas. El resultado alimenta un modelo de datos dimensional (Gold) listo para reporting o toma de decisiones de optimizaciГіn energГ©tica.

DiseГұado para escalar de 1 a N instalaciones sin cambios en el cГіdigo: aГұadir un cliente nuevo al Excel de origen es suficiente.

## 04 В· Funcionalidades principales

- вң… **Arquitectura Medallion completa** вҖ” Bronze вҶ’ Silver вҶ’ Gold con trazabilidad de linaje en cada capa
- вң… **Idempotente por diseГұo** вҖ” reejecutar el pipeline no duplica registros (upsert con clave compuesta en Silver y Gold)
- вң… **Motor fГӯsico PV de alta fidelidad** вҖ” irradiancia Haurwitz + Kasten-Czeplak + descomposiciГіn Erbs + modelo Faiman para temperatura de cГ©lula
- вң… **Resiliencia ante fallos parciales** вҖ” REE sin publicar precios devuelve `PARTIAL SUCCESS`, no aborta el pipeline
- вң… **Multi-cliente escalable** вҖ” el pipeline itera sobre todos los clientes activos; aГұadir uno no requiere cambios de cГіdigo
- вң… **AuditorГӯa automГЎtica** вҖ” cada ejecuciГіn persiste estado, duraciГіn, filas procesadas y errores en `etl_metadata`
- вң… **Tarifas elГ©ctricas espaГұolas** вҖ” clasificaciГіn automГЎtica P1/P2/P3/P6 con festivos nacionales y franjas horarias oficiales

## 06 В· InstalaciГіn y configuraciГіn

**Prerequisitos:** Python 3.11+, pip, acceso a las APIs de REE y OpenWeatherMap.

```bash
# 1. Clonar el repositorio
git clone https://github.com/tu-usuario/sunsaver.git
cd sunsaver

# 2. Crear entorno virtual e instalar dependencias
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 3. Configurar variables de entorno
cp .env.example .env
# Editar .env con las claves de API y rutas opcionales
```

Las variables de entorno necesarias estГЎn documentadas en `.env.example`. Las dos imprescindibles son `WEATHER_API_KEY` (OpenWeatherMap) y opcionalmente `DB_PATH` si se quiere un SQLite fuera del directorio por defecto.

## 10 В· Observabilidad y errores

Los logs se escriben simultГЎneamente en consola y en `logs/sunsaver_YYYY-MM-DD.log` con rotaciГіn diaria automГЎtica. El formato es `TIMESTAMP | LEVEL | MODULE | MESSAGE`, lo que facilita el filtrado con `grep` o cualquier agregador de logs.

Ante un fallo en una tarea individual, el orquestador **no aborta**: marca la tarea como `error` en el manifest y continГәa. Si un stage completo falla (todos los pasos del stage devuelven `False`), el pipeline se detiene con `FAILED AT STAGE N`. En cualquier caso, la ejecuciГіn se registra en `etl_metadata` con estado `PARTIAL SUCCESS` o `FAILED`, duraciГіn y resumen del error.

Los archivos Bronze (chmod 444) son **inmutables** una vez escritos: garantГӯa de que el dato raw original nunca se modifica, solo se vuelve a procesar desde Silver hacia arriba.

## 11 В· Decisiones de diseГұo

**Arquitectura Medallion sobre un data warehouse convencional** вҖ” La separaciГіn Bronze/Silver/Gold permite reejecutar cualquier transformaciГіn sin perder el dato original. Con un DWH clГЎsico habrГӯamos necesitado tablas de staging adicionales y lГіgica de rollback.

**SQLite sobre PostgreSQL para la capa analГӯtica** вҖ” El volumen actual (decenas de clientes, horizonte de 5 dГӯas) no justifica la complejidad operacional de un servidor. SQLite permite entregar el proyecto completo como un Гәnico fichero portable; la migraciГіn a PostgreSQL es trivial porque SQLAlchemy abstrae el dialecto.

**Motor fГӯsico propio (pvlib) sobre APIs de terceros de predicciГіn solar** вҖ” El cГЎlculo determinista garantiza reproducibilidad completa: dado el mismo input meteorolГіgico, el output es idГ©ntico. Una API externa aГұadirГӯa latencia, coste y una caja negra en el modelo.

**Manifests JSON como cola de trabajo en Bronze** вҖ” Evita una dependencia de infraestructura (Redis, Celery, Airflow) manteniendo la capacidad de reintentar tareas fallidas individualmente. La soluciГіn es suficientemente robusta para el volumen actual y trivial de entender.

**Roadmap:**
- Scheduler con APScheduler o cron para ejecuciГіn autГіnoma diaria
- ExportaciГіn de resultados a Parquet para integraciГіn con herramientas BI (Power BI, Metabase)
- Soporte multi-zona horaria para instalaciones fuera de EspaГұa peninsular
- ContenerizaciГіn con Docker para despliegue reproducible
